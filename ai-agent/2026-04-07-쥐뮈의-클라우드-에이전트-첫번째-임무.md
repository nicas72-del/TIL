### Context
클라우드 기반의 지능형 에이전트를 활용하여 다양한 업무를 자동화할 수 있는 환경이 증가하고 있습니다. 특히, 정확한 결과를 얻기 위해 프롬프트 최적화 전략이 중요합니다. 이번 문서는 에이전트의 처음 임무를 시작하는 데 있어 꼭 알아야 할 프롬프트 최적화 전략을 설명합니다.

### Core
프롬프트 최적화의 핵심 전략은 크게 세 가지로 나눌 수 있습니다.

* **에이전트 특화**: 각 에이전트의 역할에 맞는 최적화된 프롬프트와 도구를 설정합니다. 이를 통해 에이전트는 보다 효율적이고 정확한 업무 수행이 가능합니다.

* **API 사용 전략 관리**: 여러 에이전트 팀에서 API 사용을 전략적으로 관리함으로써 자원 낭비를 최소화합니다. 필요할 때만 API 요청을 하도록 설계하는 것이 중요합니다.
  
* **워크플로우 모니터링 및 개선**: 에이전트 수행 과정 중의 모든 워크플로우를 지속적으로 모니터링하고 필요한 개선을 통해 최적화합니다.

### Insight
이러한 전략을 구현함으로써 에이전트의 효율성과 준비성이 크게 향상될 수 있습니다. 특히, 에이전트 특화와 API 사용의 전략적 관리가 클라우드 기반 시스템에서의 비용 절감과 성능 향상에 기여할 것입니다. 지속적인 모니터링을 통한 워크플로우 개선은 모든 시스템의 성과를 장기적으로 향상시키는 데 필수적입니다.
